## Overview

The **DFR1216Service** provides integration with the DFRobot Unihiker Expansion Board (DFR1216), which adds servo and motor control capabilities to the K10 board. This service exposes both servo control and DC motor control via OpenAPI-compliant REST endpoints.

## Hardware Overview

### DFR1216 Expansion Board Features
- **6 Servo Channels**: PWM-controlled servo outputs (channels 0-5)
- **4 DC Motor Channels**: Dual H-bridge motor driver (motors 1-4)
- **I2C Communication**: Connects to ESP32-S3 via I2C bus
- **Power Management**: Separate power input for motors/servos
- **Voltage**: 6-12V DC input for high-power actuators

### Pin Configuration
- **I2C SDA**: Configured in UNIHIKER_K10 library
- **I2C SCL**: Configured in UNIHIKER_K10 library
- **I2C Address**: Default 0x10 (configurable)

## Features

### Servo Control
- **6 Independent Channels**: Control up to 6 standard RC servos
- **Angle Control**: 0-180° for standard servos, 0-360° for continuous rotation
- **PWM Generation**: Hardware PWM via PCA9685 chip
- **Precise Positioning**: High-resolution angle control

### Motor Control
- **4 DC Motors**: Bidirectional control with speed adjustment
- **Speed Range**: -100 (full reverse) to +100 (full forward)
- **H-Bridge Driver**: Built-in motor driver circuitry
- **Independent Control**: Each motor controlled separately

### REST API
- **OpenAPI 3.0 Compliant**: Full API specification available
- **Settings Persistence**: Save/load configuration via SettingsService
- **JSON Responses**: Standard response format for all endpoints

## Usage Examples

### Basic Initialization

```cpp
#include "services/DFR1216Service.h"

DFR1216Service expansion_service;

void setup() {
    // Initialize the service
    if (!expansion_service.initializeService()) {
        Serial.println("Failed to initialize DFR1216");
        return;
    }
    
    // Start the service
    if (!expansion_service.startService()) {
        Serial.println("Failed to start DFR1216");
        return;
    }
    
    // Register HTTP routes
    expansion_service.registerRoutes();
}
```

### Direct Servo Control

```cpp
// Center servo 0
expansion_service.setServoAngle(0, 90);

// Rotate servo 3 to 180 degrees
expansion_service.setServoAngle(3, 180);

// Continuous rotation servo full speed
expansion_service.setServoAngle(5, 0);  // or 180, depending on direction
```

### Direct Motor Control

```cpp
// Motor 1 forward at 50% speed
expansion_service.setMotorSpeed(1, 50);

// Motor 2 reverse at full speed
expansion_service.setMotorSpeed(2, -100);

// Stop motor 3
expansion_service.setMotorSpeed(3, 0);

// All motors forward
for (uint8_t i = 1; i <= 4; i++) {
    expansion_service.setMotorSpeed(i, 75);
}
```

### Python Control Script

```python
import requests
import time

BASE_URL = "http://192.168.4.1/api/DFR1216/v1"

def set_servo(channel, angle):
    response = requests.post(
        f"{BASE_URL}/servo",
        json={"channel": channel, "angle": angle}
    )
    return response.json()

def set_motor(motor, speed):
    response = requests.post(
        f"{BASE_URL}/motor",
        json={"motor": motor, "speed": speed}
    )
    return response.json()

# Sweep servo
for angle in range(0, 181, 10):
    set_servo(0, angle)
    time.sleep(0.1)

# Motor forward then reverse
set_motor(1, 100)
time.sleep(2)
set_motor(1, -100)
time.sleep(2)
set_motor(1, 0)
```

### Connection Checklist
1. ✅ I2C lines (SDA/SCL) connected to K10 board
2. ✅ Logic power (3.3V/GND) from K10 board
3. ✅ Motor/servo power (6-12V) to VIN terminal
4. ✅ Common ground between K10 and power supply
5. ✅ Servos connected with correct polarity
6. ✅ DC motors connected to motor terminals

### Power Requirements
- **Logic Power**: Provided by K10 board (3.3V @ <100mA)
- **Motor/Servo Power**: External 6-12V supply
  - Each servo: ~500mA peak
  - Each DC motor: 500mA-2A depending on load
  - **Important**: Do NOT power servos/motors from K10 board

## Best Practices

1. **Power Safely**: Use adequate external power supply for motors
2. **Test Individually**: Test each servo/motor before complex sequences
3. **Emergency Stop**: Implement stop command in control code
4. **Limit Speed**: Start with low speeds when testing motors
5. **Check I2C**: Verify I2C communication before reporting hardware failure
6. **Calibrate Servos**: Note actual angle ranges for your specific servos

## Troubleshooting

### Servos Not Moving
- ✓ Check external power supply connected and enabled
- ✓ Verify servo connectors (signal, +, -)
- ✓ Test with known-good servo
- ✓ Check I2C communication via serial logs
- ✓ Verify angle within servo's physical range

### Motors Running Wrong Direction
- Swap motor terminal connections, or
- Invert speed value in code: `setMotorSpeed(1, -speed)`

### I2C Errors
- ✓ Check SDA/SCL connections
- ✓ Verify 3.3V logic power
- ✓ Check I2C address matches hardware jumpers
- ✓ Look for bus conflicts with other I2C devices
- ✓ Add pull-up resistors if needed (usually not required)

### Jittery Servo Movement
- ✓ Use adequate power supply current rating
- ✓ Add capacitors near servo power inputs
- ✓ Check for loose connections
- ✓ Reduce update frequency

## Performance Considerations

### Update Rates
- **Servo update**: Max ~50Hz (20ms period)
- **Motor update**: Can be higher, limited by I2C speed
- **I2C bus**: 100kHz or 400kHz depending on configuration

### Multiple Operations
When controlling multiple servos/motors:
```cpp
// Batch updates for efficiency
expansion_service.setServoAngle(0, 90);
expansion_service.setServoAngle(1, 90);
expansion_service.setServoAngle(2, 90);
// Single I2C transaction per call
```
