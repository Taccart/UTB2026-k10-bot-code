## Overview

The **RemoteControlService** provides remote control functionality for the K10 Bot via UDP commands. It acts as a high-level interface that translates remote control commands (like "forward", "turn_left") into servo control actions, enabling remote operation of the robot.

## Integration with UDPService

The RemoteControlService uses the **message handler registration** pattern:

1. During `startService()`, it registers a handler function with UDPService
2. UDPService calls this handler for every received UDP packet
3. Handler returns `true` if it processed the command, `false` otherwise
4. Multiple handlers can coexist (handler chain pattern)

### Handler Registration

```cpp
bool RemoteControlService::startService()
{
    handler_id = udp_service->registerMessageHandler(
        [this](const std::string& msg, const IPAddress& ip, uint16_t port) {
            return this->handleMessage(msg, ip, port);
        }
    );
    
    return (handler_id >= 0);
}
```

## Error Handling

### Missing Dependencies
If UDPService or ServoService pointers are null:
- `initializeService()` returns `false`
- Error logged: `"Remote Control: missing required services"`

### Handler Registration Failure
If UDPService handler registration fails:
- `startService()` returns `false`
- Error logged: `"Remote Control: failed to register UDP handler"`

### Unknown Commands
- Commands not in the recognized list are ignored
- Handler returns `false` to allow other handlers to process
- No error logged (allows for command chaining)

## Best Practices

1. **Initialize dependencies first**: Always start UDPService and ServoService before RemoteControlService
2. **Test commands individually**: Verify each command works before scripting sequences
3. **Add timeouts**: Implement client-side timeouts for command acknowledgment
4. **Calibrate servos**: Ensure servo angles match your robot's physical configuration
5. **Emergency stop**: Always implement a reliable stop command
6. **Signal strength**: Test UDP reliability at expected operating distances

## Troubleshooting

### Commands Not Working
- Check UDP service is running: `udp_service.getStatus()`
- Verify network connectivity
- Monitor serial output for error messages
- Test with simple commands first ("stop", "forward")

### Servo Not Moving
- Verify ServoService is initialized
- Check servo channel numbers match physical connections
- Test servos directly via HTTP API first
- Check power supply to servos

### Intermittent Response
- UDP is unreliable by design - implement retries on client
- Check WiFi signal strength
- Reduce command frequency
- Monitor for dropped packets
